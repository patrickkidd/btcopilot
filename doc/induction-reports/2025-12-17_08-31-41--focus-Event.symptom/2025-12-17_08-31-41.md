## Recommendations

⚠️ **Modest improvement (2.7% symptom F1)**. The focused metric improved from 0.000 to 0.027, which is meaningful but still very low.

**Root Cause Analysis**: The low symptom F1 is primarily due to **event matching failures**, not symptom coding issues. Event matching requires:
- Description similarity >= 50%
- Kind match (exact)
- Person ID match (after ID mapping)
- Date within tolerance

The AI's event descriptions are often too long or combine multiple issues, causing fuzzy match to fail even when the symptom coding is correct. For example:
- GT: "Trouble sleeping" (16 chars)
- AI: "Having trouble sleeping and feeling really anxious" (48 chars)
- Similarity: 0.477 (below 0.5 threshold) → No match → Symptom F1 = 0

**Recommendations for further improvement**:
1. Consider lowering `DESCRIPTION_SIMILARITY_THRESHOLD` from 0.5 to 0.4 in f1_metrics.py
2. Add more explicit examples showing short, focused event descriptions
3. Consider semantic similarity (embeddings) instead of fuzzy string matching
4. The GT may need review - some GT event descriptions are very short while AI naturally produces longer descriptions
