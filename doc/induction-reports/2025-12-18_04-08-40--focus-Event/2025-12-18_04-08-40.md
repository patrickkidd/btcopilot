## Root Cause Analysis

The low events_f1 is caused by multiple compounding factors:

1. **ID Mismatch Brittleness**: Event matching requires ALL person links (person, spouse, child, relationshipTargets, relationshipTriangles) to match EXACTLY after ID resolution. Even one wrong ID causes event non-match.

2. **Sparse GT People**: Many GT events reference people that the AI doesn't extract at all. Without the person in both AI and GT extractions, the ID mapping fails.

3. **Description Variance**: GT descriptions are often short/generic ("New Event", "Was scrutinizing") while AI produces longer descriptions. While similarity threshold is lenient (0.4), this creates variance.

4. **LLM Stochasticity**: Results varied between 0.135-0.162 across runs with same prompts.

## Recommendations

**Events F1 improved modestly (10%) but remains below 0.20 threshold.**

Root causes require architectural changes beyond prompt tuning:

1. **Relax event matching criteria**: Consider matching events by (kind + description_sim + date) WITHOUT requiring exact person ID matches. Person assignment could be evaluated separately.

2. **Improve people extraction**: Events F1 depends on people F1 because of ID resolution. Focus on extracting ALL mentioned people first.

3. **GT quality review**: Some GT descriptions are generic ("New Event") which hurts description matching.

4. **Consider hierarchical metrics**: The hierarchical SARF F1 metrics (detection_f1, value_match_f1) may be more appropriate for measuring progress.
