## Key Changes Made

The final prompt changes that were kept:

1. **SECTION 2 (DATA_EXTRACTION_PROMPT)**: Added MANDATORY TITLE PRESERVATION rule with explicit examples:
   ```
   - **MANDATORY TITLE PRESERVATION**: When a person is mentioned with a title or
     honorific, you MUST include it in the name field EXACTLY as spoken:
     - "Aunt Linda" → name: "Aunt Linda" (NOT "Linda")
     - "Uncle Tom" → name: "Uncle Tom" (NOT "Tom")
     - "Dr Brezel" → name: "Dr Brezel" (NOT "Brezel")
     - "Grandma Rose" → name: "Grandma Rose" (NOT "Rose")
     **This is non-negotiable. Stripping titles is a critical error.**
   ```

2. **SECTION 3 (DATA_EXTRACTION_EXAMPLES)**: Refined the RULE in PERSON_NAME_PRESERVE_TITLES example:
   ```
   **RULE**: Use the EXACT name format the user provides, including ALL titles/honorifics:
   - "Aunt Linda" → "Aunt Linda" (NOT "Linda")
   - "Uncle Tom" → "Uncle Tom" (NOT "Tom")
   - "Dr Smith" → "Dr Smith" (NOT "Smith")
   - "Grandma Rose" → "Grandma Rose" (NOT "Rose")
   Stripping titles is a CRITICAL ERROR that will be penalized.
   ```

## Key Learning

**Adding new examples or expanding rules consistently hurt F1.** Only small refinements to existing content improved performance. This suggests:

- The prompt is at or near optimal length/complexity for Gemini 2.0 Flash
- Additional examples may confuse the model or cause over-application of patterns
- Targeted refinement of existing rules is more effective than adding new rules

## Recommendations

⚠️ **F1 improved 1-5% (actually ~48% for people_f1)**: Strong improvement on focused metric. Commit changes to production.

**Remaining issues not addressed:**
- Possessive patterns ("Marcus' Mother", "User's Father") - attempted but hurt F1
- Non-family person extraction ("Helen Land", "Woman on bus") - attempted but hurt F1
- Role-based references ("The Baby", "The Court") - not successfully addressed

These may require schema changes or different prompt architecture rather than incremental tuning.
