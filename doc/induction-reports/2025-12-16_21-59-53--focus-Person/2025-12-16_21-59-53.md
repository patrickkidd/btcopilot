## Error Patterns Addressed

1. **Possessive Naming Pattern**: AI was extracting "Mom" instead of "Sarah's Mom" when speaker name was known. Added explicit guidance and example showing speaker-prefixed naming.

2. **Title/Punctuation Formatting**: AI was adding periods to "Dr" ("Dr. Brezel" vs "Dr Brezel"). Added stronger character-exact matching rules and examples.

3. **Generic vs Named Deduplication**: AI was extracting both "Doctor" and "Dr Brezel" for the same person. Added rule to extract only the named version.

4. **Named Person Extraction**: Added explicit examples for extracting all named individuals even when mentioned in passing (e.g., "Watch out for Helen Land").

## Key Changes Made

1. **Added possessive naming rule** in SECTION 2 with examples:
   - Speaker is "Sarah" and says "my mom" → "Sarah's Mom"
   - Speaker is "Marcus" and says "my girlfriend" → "Marcus' Girlfriend"
   - Unknown speaker → "User's Mother"

2. **Added possessive naming example** in SECTION 3:
   - Shows wrong output ("Mom") vs correct output ("Sarah's Mom")
   - Emphasizes speaker name prefix requirement

3. **Strengthened exact character matching** rules:
   - "Dr Brezel" → "Dr Brezel" (never "Dr. Brezel")
   - "The Baby" → "The Baby" (preserve capitalization)
   - Explicit guidance: copy character-for-character

3. **High variance**: LLM responses show 0.071 variance (0.629-0.700) due to stochastic nature.
