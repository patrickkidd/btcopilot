## Error Patterns Addressed

1. **Scene-detail over-extraction**: AI extracted physical actions ("slammed laptop", "made a drink") as clinical events. Fixed with explicit scene-detail suppression hint.
2. **Missing birth events**: AI produced zero births despite ages mentioned in transcript. Partially addressed with birth reminder and age calculation formula, but non-deterministic — Gemini sometimes ignores it.
3. **Relationship type confusion**: projection vs overfunctioning, inside vs conflict. Added explicit disambiguation rules.
4. **Event duplication**: Multiple similar events for same person/pattern/time period. Added deduplication guidance.

## Iteration History

### Iteration 1 — V1: Consolidation Rules
- **Change**: Added consolidation rules, target event counts, birth event rules after transcript
- **Rationale**: Reduce over-generation (48 AI events vs 29 GT)
- **F1**: 0.429 on disc 50 (baseline 0.416)
- **Outcome**: Reverted — model completely ignored rules. Still extracting scene-level micro-events.

### Iteration 2 — V2: Aggressive Pre-Transcript Rules
- **Change**: Rules placed BEFORE transcript with CRITICAL framing. "DO NOT APPLY" per-statement rules.
- **Rationale**: Post-transcript rules ignored; try pre-transcript with stronger framing
- **F1**: 0.344 on disc 50 (below baseline)
- **Outcome**: Reverted — over-consolidation merged distinct clinical events. TP dropped from 15 to 11.

### Iteration 3 — V3: Episode Granularity
- **Change**: Introduced "episode granularity" concept. Post-transcript checklist.
- **Rationale**: Middle ground between V1 (too weak) and V2 (too aggressive)
- **F1**: 0.286-0.344 on disc 50 (high variance, 8-13 TP)
- **Outcome**: Reverted — consolidation changes descriptions from verbatim to clinical labels, breaking fuzzy matching.

### Iteration 4 — V4: "IGNORE" Override Mode
- **Change**: Complete rewrite. "IGNORE the per-statement extraction approach above."
- **Rationale**: Override 1770 lines of per-statement training with explicit full-extraction steps
- **F1**: ~9 TP on disc 50 (count control worked but wrong events selected)
- **Outcome**: Reverted — IGNORE destroys useful per-statement event detection. Net negative.

### Iteration 5 — V5: "Follow BUT Override"
- **Change**: "Follow the extraction rules above BUT with these critical overrides." WRONG/RIGHT examples.
- **Rationale**: Softer override to preserve per-statement training
- **F1**: 76 events on disc 50 (explosion). 15 TP but terrible precision.
- **Outcome**: Reverted — without IGNORE, model reverts to per-statement behavior. 1770 lines overpower 80 lines.

### Iteration 6 — V6: Person-Centric + "DO NOT APPLY"
- **Change**: Person-centric extraction (1-3 episodes per person). Explicit birth age calculation. After transcript.
- **Rationale**: Compromise framing + person-centric approach
- **F1**: 28 events, 10 TP, zero births
- **Outcome**: Reverted — person-centric didn't help. Birth events remain non-deterministic.

### Iteration 7 — V7: Minimal Intervention
- **Change**: Near-original prompt + 4 targeted adjustments
- **Rationale**: Hypothesis that original simple prompt works reasonably well
- **F1**: Inconsistent (discovered baseline also varies due to non-determinism)
- **Outcome**: Superseded by V8

### Iteration 8 — V8: Quality Hints Only
- **Change**: Original "extract everything" structure + QUALITY GUIDELINES section with 4 lightweight hints (scene detail suppression, birth reminder, relationship types, deduplication). No count control or override framing.
- **Rationale**: Minimal intervention — per-statement training is useful, just add quality guardrails
- **F1**: 0.317 (run 1), 0.346 (run 2) across all 6 discussions
- **Outcome**: Kept and extended

### Iteration 9 — V9: V8 + Soft Calibration (FINAL)
- **Change**: Added "A typical full discussion yields 15-30 total events" calibration hint
- **Rationale**: Gentle count guidance without hard limits
- **F1**: 0.367 / 0.313 / 0.325 across 3 full runs (avg 0.335)
- **Outcome**: Kept as final version

### Matching Threshold Experiment
- **Change**: Raised `DESCRIPTION_SIMILARITY_THRESHOLD` from 0.4 to 0.5 in f1_metrics.py
- **Finding**: False matches score 0.41-0.49, true matches score 0.51+. Clean gap exists.
- **Result**: Events F1 dropped to 0.200 — false matches were inflating TP count. Removing them just increased FN.
- **Outcome**: Reverted. The 0.4 threshold is a pragmatic compromise.

## Key Lessons Learned

### What Worked

1. **Minimal intervention outperforms aggressive rewriting.** The per-statement training (1770 lines) encodes useful event extraction behavior. Light quality hints layered on top work best. V8/V9's quality guidelines approach was the only one that produced sustained improvement.

2. **Scene-detail suppression with concrete examples.** Listing specific scene details ("slammed door", "made a drink", "stared at TV") as what NOT to extract was effective and non-destructive.

3. **Soft calibration (range, not target).** "15-30 total events" as a sanity check, not a hard target. Doesn't force the model to cut events to a number.

### What Failed

1. **Aggressive consolidation / count control.** Every approach that tried to reduce event counts (V1-V4, V6) killed TP proportionally to FP. The model doesn't know which events to keep.

2. **"IGNORE" / "DO NOT APPLY" framing.** Destroys useful per-statement training. The 1770 lines of examples are the model's primary event detection capability.

3. **Pre-transcript rule placement.** V2 showed that rules before the transcript are actually less effective than after (recency bias matters, but 1770 lines of examples still dominate).

4. **Person-centric extraction.** V6's "1-3 most significant episodes per person" didn't improve event selection quality.

5. **Explicit birth instructions.** Birth event generation is non-deterministic in Gemini 2.5 Flash regardless of prompt specificity. Instructions ranging from simple ("create birth events") to detailed ("Calculate year as 2026 minus stated age, typically find 2-5, if zero re-read") all produce the same inconsistent results.

6. **Raising similarity threshold to 0.5.** Theoretically correct (eliminates false matches) but hurts measured F1 because false TPs were being counted.

### Fundamental Constraints Discovered

1. **Description style mismatch is the binding constraint.** GT uses speaker's verbatim words ("went into a shell"). AI uses clinical summaries ("emotionally withdrawn"). Any prompt change that increases clinical abstraction will hurt matching regardless of clinical accuracy.

2. **1770 lines of per-statement examples dominate model behavior.** Full-extraction context (~50 lines) cannot override this. The effective strategy is to work WITH the per-statement training, not against it.

3. **LLM non-determinism is ~10-15%.** F1 varies +/-0.05-0.10 between identical runs. Single-run measurements are unreliable. Multi-run averaging is necessary for any signal.

## Recommendations

- **V9 should be kept (committed).** The improvement is modest (~11% average) but consistent, and no regressions were introduced.
- **Further full-extraction prompt improvements are unlikely to yield >5% additional improvement** without addressing the matching algorithm (embedding similarity) or GT description quality.
- **Next high-impact work should target**: (1) embedding-based description similarity, (2) GT description cleanup, (3) multi-run averaging in the evaluation harness.
