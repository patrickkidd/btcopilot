# T3-4: SARF extraction should show direction format

**Status:** Fix applied
**Date:** 2026-02-20

## Problem
PDPEventCard.qml displayed SARF variables in a 3-line format (ColumnLayout with label on one line, value on next line). Should show compact "Label: Value" single-line format like "Symptom: Up".

## Fix
Collapsed each SARF variable display from ColumnLayout with two Text elements into a single Text element with colon-separated format. Applied to all four variables:
- Symptom: `"Symptom: " + personalApp.variableLabel(eventData.symptom)`
- Anxiety: `"Anxiety: " + personalApp.variableLabel(eventData.anxiety)`
- Relationship: `"Relationship: " + personalApp.relationshipLabel(eventData.relationship)`
- Functioning: `"Functioning: " + personalApp.variableLabel(eventData.functioning)`

Preserved existing color coding via `variableColor()` and visibility conditions.
