## Key Decisions

### Symptom Definition Narrowed
Sensitivity, reactivity, and behavioral characteristics are **anxiety, not symptom.** Symptom = functional breakdown / failure mode — the point where anxiety exceeds functioning capacity. Don't widen symptom to include presenting behaviors or personality traits. Kathy will revert from her broadened usage.

### Baseline vs. Shift Distinction
"Always been X" = baseline characterization. Code once with unknown date certainty. **Shifts require timestamps.** Exception: evidence of a baseline changing permanently at a specific time ("that shift became permanent") IS a timestamped shift. Laura: "In order to put functioning down, it has to be a time-stamped event."

### Structural Events ≠ SARF Shifts
Birth, death, marriage, divorce, bonded, moved, adopted = structural (family configuration changes). SARF shifts = functional (how the organism responds). Track both but as separate data entries. The add-event form enforces this: one type per entry.

### Functional Facts Are Not Shifts
Occupation, education, health, location are Bowen's multigenerational functioning markers. Don't code as shift events. Currently no proper home in the coding system — acknowledged data model gap for future work.

### Don't Code Functioning for Infants
Colic = valid symptom. Functioning down = not meaningful for infants (no PFC).
