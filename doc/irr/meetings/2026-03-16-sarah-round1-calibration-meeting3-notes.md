## Key Decisions

### Cumulative Timeline is the Primary Comparison Unit
Patrick concluded that cumulative timeline comparison matters more than statement-by-statement agreement. Statement-by-statement is needed for data entry workflow, but "all that matters is we came up with the same timeline." This partially resolves the open measurement approach question.

### Blind Coding Protocol
Group agreed: code first, then use the AI lit review tool after. Don't use lit review as input during blind coding — treat the AI tool like another coder whose output shouldn't influence yours. Formal protocol for research use to be decided later.

### Deduplication — Laura Aligned
Laura (absent from Meeting 2) confirmed understanding of the deduplication rule: one delta per construct per time period, don't re-enter when re-mentioned in later statements. Applies across all statements, not just within one turn.
