Data Created

Generating a synthetic discussion creates the following records:

  • 1 Diagram - Family diagram owned by the selected user
  • 1 Discussion - Conversation thread linked to the diagram
  • N Statements - One per conversation turn (user message + AI response)
  • N pdp_deltas - AI extractions per statement (if extraction enabled)

Discussions are marked synthetic=True and can be filtered in the Admin dashboard.

About Synthetic Discussions

Synthetic discussions are generated using LLM-simulated personas that respond naturally based on their defined background, traits, and presenting problem.

Two purposes:

  • Ground Truth Corpus: Generate and persist conversations, then code the extractions as ground truth for F1 metrics.
  • Prompt Testing: Test conversation quality and coverage metrics to validate prompt changes.

Workflow:

  1. Generate a synthetic discussion (with extraction enabled)
  2. Review the conversation in the audit UI
  3. Edit and correct the AI extractions
  4. Approve as ground truth
